Developer Evaluation

Calculate stuff!

Instructions
  • Enter a string of number into a text box, delimited by ","
    • Example: 1, 2, 3, 4, 5
  • When user hits Enter key or hits the "Calculate" button
    • As per the definitions at https://www.mathisfun.com/data/central-measures.html, calculate the following using all the numbers
      • Average/Mean
      • Median
      • Mode
    • The calculation must be done on the backend service, so the web project must call the service project to get the answers.
    • Display the results somewhere on the page
  • Create front end unit tests and backend unit tests. Front end unit tests should be runnable and passable by running 'ng test'. Backend unit tests should be runnable by running 'dotnet test' on the .UnitTests.csproj project.
  • Recommended to use Karma/Jasmine for the frontend unit tests (already installed libraries to web project).
  • Recommended to use NUnit for the backend unit tests (already installed NuGet packages to service project).
Project Setup
  • Using Visual Studio 2017 and .NET Core 2.1
  • Using Angular 6 in VSCode
  • Open the web solution in VSCode and the service solution in Visual Studio 2017
  • Restore NuGet packages in the service project and run 'npm install' in the web project
  • Run in Debug for the service application (should open a browser window)
  • Run the web application by executing 'npm start'
Extra Points
  • Add a table (or how ever many you think you'll need) to the database project and connect up the table to the service project so that when the user calculates something it logs important infomration like the timestamp when they clicked the button, the inputs, and the output to the table(s). If you do this then we should be able to deploy/publish the database .dacpac to a SQL server for when we test it out.
  • Enhance the user experience in some way.
  • Add some routes to the web project which directs the user to different types of functionality in the app.
  • Add styles and make the website responsive
  • Add some sort of documentation to the web project and/or the service project (like Swagger).
  • Add anything else extra that you think might add to the application in a positive way!
